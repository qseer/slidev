# 项目与项目式学习

## 什么是项目？

大家可以思考一下,这里我也给大家一些案例。比如一个公司老板说,最近又给大家拉了几个项目,大家要好好加油,努力完成,等项目完成了，我们给大家发奖金，犒劳大家这一段时间的辛苦付出,这是一个情景。那么在这个情景里面,项目它可能更多是一个赚钱的东西。它是维持公司生计的必要营收来源。也可以说，对于公司而言，没有项目就没有存活。
我们再回到大家平时的学校情景中,比如说学校想要举办一场运动会,或者是校园文化节,那么此时学校想要主办筹办这样一场活动的话,其实这也是一个项目。那么有的学校呢,他可能会把很多相关的筹备工作交给学生来完成,那么大家在做这件事的时候,其实就是在完成一个项目。
或者我们再看一个离大家生活更近的情景，大家平时在班级里做黑板报，大家做的这个黑板报也可以说是一个项目。
...

所以通过我们上面讲到的案例,我们大家再来一起回顾一下,究竟什么是一个项目呢?它可以是一个公司的研发产品,它也可以是一场活动的策划,它也可以是大家平时的一幅画,一幅作品等等,这些都可以叫做项目。那么我们刚刚看了几个项目的案例以后,大家能不能从其中归纳出一些共同点呢?我们如何对一个项目来去进行一个自己的定义呢?
根据老师自己的一个通俗的理解和狭义的定义,项目它是针对某一个特定人群，解决他们某一个特定问题，满足他某一个特定需求的“东西”。
那么通过这个定义来看,大家说项目它是一个客观的,有标准答案的问题吗?其实往往是不是的,很多时候项目它并不是一个有着标准答案的事情。我们在评判一个项目好坏的时候,往往也没有十分精确可以量化的评分标准。不像大家平常的期中考试、期末考试，大家学得如何？考一次试就知道了，分数可以是一个十分客观精确的评判指标。但是去给一个项目进行十分精准精确的打分,很多时候这是做不到的。那我们评判一个项目的一个标准很多时候是偏主观性的,有的人他可能觉得这个项目做得好,有的人可能觉得这个项目做得非常差,这些都是有可能的。


那么之前听韩老师讲,大家已经学了一些Python的代码基础了,我觉得这对于我们这个项目学习的课程的开展是有极大的一个帮助的。大家距离一个完整的项目,从代码实现上来讲,就差一个展示的窗口了。这个展示的窗口就是网页。而网页就是我们接下来几周想要带大家一起做的一个小项目。

我们这里用一个形象的比喻来去类比一下项目的话呢,可以这么理解,现在我们所学习的python代码,它相当于是一个产品机器内部的这个构造原理和运行机制,这个是底层的核心,但是我们光有这个东西,它确实可以工作了,但是它并不方便于我们人机进行交互,那么这个时候我们往往会给这个机器再套上一层外壳,添加一些按钮来便于我们去操作它,那我们所套的这层外壳呢,就是我们现在想要制作的网页了。

## 为什么要做项目？

python 编写出来的程序，是驱动的底层引擎，可以让车跑，但只是一个骨架。只有添加上舒适的座椅、遮风的外壳，这才是一个可交付给用户使用的真实产品。我们现在所做的网页，就是给你的 Python 代码加上「外壳」，加上舒适的用户体验和人机交互的接口，让一些你的人群和受众真的能使用它，并从中受益。

# 网页项目案例

这里我们来看几个直观的例子,大家可以直观感受，什么是一个网页项目，也都是一些经典的网页项目。

【电子木鱼案例】
比如说,大家可能之前会见到过一些电子木鱼的网页小游戏。这个小游戏老师也是非常喜欢的,就是每次点击一下它,它会告诉我功德加一,或者是财富加一等等,非常的有趣。 那么我们想要实现这么一个小玩意儿,它的一个这个实现原理和实现方式是怎么样做的呢?如何一步一步把它做出来?大家之前学过python,有没有同学可以说一说自己的想法呢?
【SSR 抽卡案例】

我们通常说一个好的游戏,它不光要有好的玩法,也就是这背后的这个程序的算法机制,底层的运行机制,但是一个好的游戏,它同时也应当具有一副好的好看的皮囊,那么网页就是能够传递这个好看的皮囊的一种方式与途径。

我们可以对比一下，如果只有 Python 程序，我们确实可以实现电子木鱼的玩法，但是你无法把它封装成一个人们可以很好、易于使用的产品。那么也许你的玩法再好，游戏再创新，但是无人问津。这个游戏也不能称作是一个成功的项目。正如我们俗话说，酒香也怕巷子深，对吧？我们想要把自己的作品和产品很好地展现并推广出去。这时候，我们需要打造一套好的人机交互界面，以及良好的信息传递系统。这个时候，网页就可以发挥作用了。
