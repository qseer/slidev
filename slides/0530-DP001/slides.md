# 线性 DP 入门

从记忆化搜索循序渐进过渡到动态规划

<br/>

四道题：方格取数 · LIS · LCS · 最大子段和

---

# 这节课要回答的问题

- DP 的「状态」是怎么想出来的？
- 「状态转移方程」凭什么能写出来？
- 为什么有时候自然的状态写不出转移？

<br/>

## 我们的策略

不直接讲 DP，而是从**暴力 DFS** 开始：

1. 写一个最朴素的递归
2. 发现重叠子问题 → 加 memo（记忆化搜索）
3. 把递归改写成 for 循环（DP）

> 这套流水线走通了，DP 的「状态」就不再是天上掉下来的

# 第一题：方格取数

## 热身 — 建立 DFS → memo → DP 流水线

---

## 题面

`n × n` 方格，每格有一个非负权值。

从 `(1,1)` 走到 `(n,n)`，每步只能 **向右** 或 **向下**。

**求路径上权值之和的最大值。**

<br/>

```
 1  3  1        路径：1 → 3 → 5 → 2 → 1
 1  5  1                          
 4  2  1        和：12（其它路径都 ≤ 12）
```

---

## 第一步：从「答案」反推「状态」

> 要求的是什么？
> → **从 `(1,1)` 走到 `(n,n)` 的最大和**

把「具体位置」换成「参数」：

```cpp
// dfs(i, j) = 从 (1,1) 走到 (i,j) 的最大和
```

<br/>

<Block type="info" title="状态是怎么来的？">

把「最终答案那句话里的具体数字」换成参数，就是状态。
所以最终答案 = `dfs(n, n)`。

</Block>

---

## 第二步：推转移

走到 `(i, j)` 之前的最后一步在哪？

- 从上面 `(i-1, j)` 走下来
- 或从左边 `(i, j-1)` 走过来

转移直接落地：

```cpp
int dfs(int i, int j)
{
    if(i < 1 || j < 1) return 0;
    if(i == 1 && j == 1) return grid[1][1];
    return max(dfs(i-1, j), dfs(i, j-1)) + grid[i][j];
}
```

跑样例：**对的**。但 `n = 1000` 时呢？

---

## 第三步：发现重叠子问题

手动展开递归树前两层（`n = 3`）：

```
                dfs(3,3)
               /        \
          dfs(2,3)      dfs(3,2)
          /    \         /    \
     dfs(1,3) dfs(2,2) dfs(2,2) dfs(3,1)
                 ↑         ↑
                 └─── 重复！───┘
```

<br/>

- 复杂度 `O(2^(2n))` — `n = 30` 就跑不动了
- 但 `dfs(2,2)` 不管被请求几次，**答案永远一样**
- 那就 **算过一次就记下来**

---

## 第四步：加 memo（记忆化搜索）

```cpp
int memo[maxn][maxn];  // 初值 -1 表示未算过（权值非负，安全）

int dfs(int i, int j)
{
    if(i < 1 || j < 1) return 0;
    if(i == 1 && j == 1) return grid[1][1];
    if(memo[i][j] != -1) return memo[i][j];
    return memo[i][j] = max(dfs(i-1, j), dfs(i, j-1)) + grid[i][j];
}
```

<br/>

复杂度：`O(2^(2n))` → **`O(n²)`** — 每个 `(i,j)` 只算一次。

---

## 第五步：改写成 DP

> 既然每个 `(i,j)` 只算一次，能不能不用递归，**主动**把所有 `(i,j)` 算出来？

观察依赖：`dp[i][j]` 只依赖 `dp[i-1][j]` 和 `dp[i][j-1]` — 都在它的左上方。

那就按 `i` 从小到大、`j` 从小到大填：

```cpp
for(int i = 1; i <= n; ++i)
    for(int j = 1; j <= n; ++j)
        dp[i][j] = max(dp[i-1][j], dp[i][j-1]) + grid[i][j];
cout << dp[n][n];
```

---
layout: default
title: 三种写法对照
---

## 三种写法是同一件事

| 暴力 DFS | 记忆化搜索 | DP |
|---|---|---|
| `dfs(i, j)` | `dfs(i, j)` + `memo` | `dp[i][j]` |
| 递归调用 `dfs(i-1, j)` | 递归调用，但查 memo | 直接读 `dp[i-1][j]` |
| 重复计算 | 由 `memo` 防重复 | 由 *填表顺序* 防重复 |
| `O(2^(2n))` | `O(n²)` | `O(n²)` |
| 自顶向下 | 自顶向下 | 自底向上 |

<br/>

<Block type="success" title="第一个心法">

**DP 数组 = 记忆化的 memo。**
所谓「DP 的状态定义」，就是「DFS 函数的返回值定义」。

</Block>

# 第二题：最长上升子序列（LIS）

## 提出心法 — 状态是被转移「逼」出来的

---

## 题面

给定长度为 `n` 的序列 `a[1..n]`，求**最长上升子序列**的长度。

子序列：可以不连续，但保持原顺序。

<br/>

```
a = [1, 5, 2, 4, 3]

LIS：1 < 2 < 3   或   1 < 2 < 4
答案：3
```

---

## 第一回合：套用 T1 的套路

> 状态怎么定义？
> → 模仿 T1，把答案里的「整体」换成「前 i 个」

```cpp
// dfs(i) = 前 i 个数的 LIS 长度
int dfs(int i)
{
    if(i == 0) return 0;
    // 选 a[i]：要接到前面某个 < a[i] 的数后面…
    //         可前面 LIS 的末尾是什么？我不知道
    // 不选 a[i]：dfs(i-1)
    return max(dfs(i-1), /* 选 a[i] */ ???);
}
```

**卡住了。** 为什么？

---
layout: statement
---

# 诊断

`dfs(i-1)` 只告诉你「前面能凑出多长的 LIS」，

**没告诉你那个 LIS 的末尾是什么**。

所以 `a[i]` 能不能接上去，根本无从判断。

<br/>

→ **状态信息量不够。**

---

## 第二回合：补足信息

### 路 A：状态多记一维「末尾值」

```cpp
// dfs(i, last) = 在前 i 个里，必须接在值 last 之后的 LIS 长度
```

可行，但 `last` 值域大 → 状态空间爆炸 ❌

<br/>

### 路 B：强制规定末尾就是 `a[i]`

```cpp
// dfs(i) = 必须以 a[i] 结尾的 LIS 长度
```

末尾被写死成已知值，转移立刻能写 ✅

---

## 转移自然出现

```cpp
int dfs(int i)
{
    int res = 1;                          // 至少能取 a[i] 自己
    for(int j = 1; j < i; ++j)
        if(a[j] < a[i])
            res = max(res, dfs(j) + 1);   // 接在以 a[j] 结尾的 LIS 后面
    return res;
}
```

<br/>

为什么这能写出来？
→ 因为状态写死了「末尾是 `a[i]`」，子问题 `dfs(j)` 的末尾就是已知的 `a[j]`，能直接比较。

---

## 注意：答案不是 `dfs(n)`！

`dfs(n)` 是「必须以 `a[n]` 结尾」的 LIS，不一定是全局最长的。

```
a = [1, 5, 2, 4, 3]
dfs(5) = 3   （以 3 结尾：1 2 3）
dfs(4) = 3   （以 4 结尾：1 2 4）
全局最大 = 3
```

<br/>

要遍历所有可能的结尾：

```cpp
int ans = 0;
for(int i = 1; i <= n; ++i) ans = max(ans, dfs(i));
```

> 「以 i 结尾」是我们为了能写转移而付出的代价。

---

## 加 memo → 改 DP

```cpp
// 记忆化：LIS 长度 ≥ 1，memo[i] == 0 即表示未算过
int dfs(int i)
{
    if(memo[i]) return memo[i];
    int res = 1;
    for(int j = 1; j < i; ++j)
        if(a[j] < a[i]) res = max(res, dfs(j) + 1);
    return memo[i] = res;
}
```

改写成 for：

```cpp
for(int i = 1; i <= n; ++i)
{
    dp[i] = 1;
    for(int j = 1; j < i; ++j)
        if(a[j] < a[i]) dp[i] = max(dp[i], dp[j] + 1);
}
```

---
layout: statement
---

# 心法

DP 的状态**不是猜的**。

先尝试最自然的状态，写转移；

写不出转移时 — 就说明状态记的信息不够。

这时候去想：**要补什么？怎么补代价最小？**

# 第三题：最长公共子序列（LCS）

## 心法复用 — 同一堵墙，不同形式

---

## 题面

给定两个字符串 `s` 和 `t`，求它们的**最长公共子序列**长度。

<br/>

```
s = "abcbdab"
t = "bdcaba"

LCS："bdab" 或 "bcba"
答案：4
```

---

## 第一回合：再套一次 T3 的套路

```cpp
// dfs(i) = s 前 i 个字符与整个 t 的 LCS
int dfs(int i)
{
    if(i == 0) return 0;
    // 选 s[i]：要在 t 里找一个相同字符接上去 — t 的哪个位置？我不知道
    // 不选 s[i]：dfs(i-1)
    return max(dfs(i-1), /* 选 s[i] */ ???);
}
```

<br/>

**和 T3 撞的是同一堵墙**，只是这次是：

> 不知道前面的 LCS **在 `t` 里用到了哪儿**。

---

## 补足信息：把 `t` 也参数化

```cpp
// dfs(i, j) = s 前 i 个字符与 t 前 j 个字符的 LCS 长度
```

<br/>

<Block type="info" title="心法的两次应用">

| | 撞的墙 | 补什么 |
|---|---|---|
| **T3 LIS** | 不知道前面 LIS 的末尾 | 加约束：「必须以 `a[i]` 结尾」 |
| **T4 LCS** | 不知道前面 LCS 在 `t` 用到哪 | 加一维参数：`t` 的前 `j` 个 |

形式不同，**起因相同** — 信息不够。

</Block>

---

## 推转移：看末尾两个字符的关系

只有两种情形：

- **`s[i] == t[j]`** ：一定可以把它俩配成 LCS 的最后一对（不会更差）
  → `dfs(i, j) = dfs(i-1, j-1) + 1`

- **`s[i] != t[j]`** ：它俩至少有一个用不上
  → `dfs(i, j) = max(dfs(i-1, j), dfs(i, j-1))`

```cpp
int dfs(int i, int j)
{
    if(i == 0 || j == 0) return 0;
    if(memo[i][j] != -1) return memo[i][j];
    if(s[i-1] == t[j-1]) return memo[i][j] = dfs(i-1, j-1) + 1;
    return memo[i][j] = max(dfs(i-1, j), dfs(i, j-1));
}
```

---

## 改写成 DP

依赖：`dp[i][j]` ← `dp[i-1][j-1]` / `dp[i-1][j]` / `dp[i][j-1]` — 都在左上方。

按 `i, j` 从小到大双重循环：

```cpp
for(int i = 1; i <= n; ++i)
    for(int j = 1; j <= m; ++j)
    {
        if(s[i-1] == t[j-1]) dp[i][j] = dp[i-1][j-1] + 1;
        else                 dp[i][j] = max(dp[i-1][j], dp[i][j-1]);
    }
cout << dp[n][m];
```

<br/>

> 和 T1 一模一样的流水线，只是状态从一维 `(i)` 变成二维 `(i, j)`。

# 第四题：最大子段和

## 反例 — 不是所有题都适合这套流水线

---

## 题面

给定长度为 `n` 的序列 `a[1..n]`（可能有负数），

**找一段连续的子段，使其和最大**。

<br/>

```
a = [-2, 11, -4, 13, -5, -2]

最大子段：[11, -4, 13]
和：20
```

---

## 试试套路：写个最自然的 DFS

> 求一段连续区间的最大和 → 自然想到 `dfs(l, r)` 枚举区间？

```cpp
// dfs(l, r) = 区间 [l..r] 的和
```

但这只是「**枚举所有子段**」的暴力，复杂度 `O(n²)`（甚至 `O(n³)` 如果不前缀和）。
而且**它根本不是 DP** — 没有重叠子问题可言。

<br/>

→ **这套流水线在这道题上跑不动。**

---
layout: statement
---

# 这道题需要「前置思考」

有些 DP 题的状态，

不是从自然递归里**演化**出来的，

而是要先想清楚 **「最优解长什么样」** 再设状态。

---

## 关键观察

最大子段一定是「**以某个位置 `i` 结尾**」的子段。

那就先解决「以 `i` 结尾的最大子段和」这个子问题，再对所有 `i` 取最大值。

<br/>

定义状态：

```cpp
// dp[i] = 以 a[i] 结尾的最大子段和
```

转移只有两种选择：

- 把 `a[i]` **续到** `dp[i-1]` 后面：`dp[i-1] + a[i]`
- **从 `a[i]` 重起一段**：`a[i]`

```cpp
dp[i] = max(dp[i-1] + a[i], a[i]);
ans = max(ans, dp[i]);
```

---

## 也能写成记忆化搜索吗？能，但绕

```cpp
// dfs(i) = 必须以 a[i] 结尾的最大子段和
int dfs(int i)
{
    if(i == 1) return a[1];
    if(vis[i]) return memo[i];
    vis[i] = true;
    return memo[i] = max(dfs(i-1) + a[i], a[i]);
}
```

<br/>

注意：`a[i]` 可负 → 不能用 `memo[i] == 0/-1` 表示未算过，要单独开 `vis[]`。

<br/>

<Block type="warning" title="对比 T3：同样是「以 i 结尾」，思路顺序却不同">

- **T3 LIS**：自然递归 → 撞墙 → 被迫加「以 i 结尾」
- **T2 最大子段和**：先观察答案结构 → 主动定义「以 i 结尾」

</Block>

# 收尾：今天学到了什么

---

## 三个心法

<br/>

<Block type="success" title="心法 1：DP 数组 = 记忆化的 memo">

DP 的「状态」就是 DFS 函数的「返回值定义」。
DP 的「转移」就是 DFS 的「递归式」。
DP 的「填表顺序」就是 DFS 的「依赖顺序」。

</Block>

<Block type="success" title="心法 2：状态被转移「逼」出来">

先写最自然的递归，写不出转移就说明状态信息不够。
这时候才去想：要补什么？怎么补代价最小？

</Block>

<Block type="success" title="心法 3：有些 DP 需要前置思考">

不是所有题都能从自然递归演化出来。
当自然递归连「最优解长什么样」都没看清时，要先观察答案结构再定义状态。

</Block>

---

## 四道题地图

| 题 | 状态来源 | 关键状态 | 维度 |
|---|---|---|---|
| **T1 方格取数** | 答案直接参数化 | `dp[i][j]` 走到 (i,j) 的最大和 | 2D |
| **T3 LIS** | 撞墙后被迫加约束 | `dp[i]` 以 a[i] 结尾的 LIS | 1D |
| **T4 LCS** | 撞墙后加一维参数 | `dp[i][j]` s 前 i / t 前 j 的 LCS | 2D |
| **T2 最大子段和** | 前置观察 + 主动定义 | `dp[i]` 以 a[i] 结尾的最大子段和 | 1D |

<br/>

> 四道题，三种「状态从哪来」的路径。后面的区间 DP、树形 DP、背包 DP 会反复用到这套思路。
